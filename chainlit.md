# Добро пожаловать в NekoChat 🐾
Твой личный пушистый ИИ-ассистент с поддержкой расширенного рассуждения, веб‑поиска и работы с PDF. Идеален для глубокого анализа информации и продуктивного общения с ИИ.

## Основные возможности

* **Расширенное рассуждение** (`use_reasoning`): позволяет модели «думать шаг за шагом» перед ответом.
* **Веб‑поиск** (`web_search`): автоматически ищет актуальную информацию в интернете.
* **Работа с PDF** (`pdf_parsing`): анализирует содержимое PDF‑файлов, включая сканы и изображения.
* **Сохранение контекста**: вся история диалога сохраняется в папке `context` для удобного редактирования и повторного использования.
* **Персонаж‑помощник**: ваш ИИ‑ассистент — элегантная и внимательная неко‑помощница, сочетающая экспертность с эмпатией.

## Настройки по умолчанию

При запуске NekoChat использует следующие настройки:

| Параметр | Значение | Описание |
|--------|----------|----------|
| `temperature` | `0.7` | Баланс между креативностью и предсказуемостью ответа. |
| `max_tokens` | `8192` | Максимальное количество токенов (слов/символов) в ответе. |
| `use_reasoning` | `True` | Включает режим пошагового рассуждения. |
| `reasoning_effort` | `medium` (Сбалансированный режим) | Уровень детализации рассуждений. |
| `reasoning_summary` | `concise` (Краткое резюме) | Формат итогового вывода после рассуждений. |
| `reasoning_max_tokens` | `0` | Лимит токенов для блока рассуждений (`0` — без ограничений). |
| `reasoning_exclude` | `False` | Определяет, исключать ли блок рассуждений из финального ответа. |
| `web_search` | `True` | Автоматически выполняет поиск в интернете по запросу. |
| `search_count` | `3` | Количество источников, которые будут найдены и проанализированы. |
| `web_engine` | `auto` (Авто) | Используемый поисковик. |
| `pdf_parsing` | `True` | Активирует парсинг и анализ PDF‑файлов. |
| `pdf_engine` | `pdf-text` (Извлечение текста) | Инструмент для обработки PDF. |
| `use_system_prompt` | `True` | Использует системный промпт для настройки поведения модели. |
| `system_prompt` | `SYS_PROMPT` | Системный промпт, задающий «личность» и правила поведения модели. |

## Доступные варианты настроек

### Поисковые системы (`web_engine`)

Определяет, какой поисковик будет использоваться для получения актуальной информации.

* **`auto` (Авто)**: система сама выберет оптимальный провайдер (может выбрать Exa).
* **`native` (Нативный)**: использует встроенную поисковую систему провайдера (недоступно для Gemini).
* **`exa` (Exa)**: использует поисковик Exa. Обратите внимание: стоимость — 2 р за поиск.

### Движки парсинга PDF (`pdf_engine`)

Определяет метод обработки PDF‑файлов.

* **`pdf-text` (Извлечение текста)**: извлекает текст напрямую из PDF. Подходит для файлов с текстовым слоем.
* **`native` (Встроенная обработка)**: использует встроенные возможности провайдера для работы с PDF.
* **`mistral-ocr` (OCR через Mistral)**: применяет технологию оптического распознавания символов (OCR) через модель Mistral. Идеально подходит для отсканированных документов и PDF с изображениями.

### Уровень усилий рассуждения (`reasoning_effort`)

Управляет глубиной и детализацией «размышлений» модели перед формированием ответа.

* **`none` (Рассуждения отключены)**: модель отвечает сразу, без внутреннего анализа.
* **`minimal` (Минимальные рассуждения)**: краткий внутренний анализ, 1–2 шага.
* **`low` (Сниженные рассуждения)**: базовый уровень рассуждений, несколько шагов.
* **`medium` (Сбалансированный режим)**: оптимальный баланс между скоростью и глубиной анализа (используется по умолчанию).
* **`high` (Детальные рассуждения)**: подробный пошаговый анализ, высокая детализация.
* **`xhigh` (Максимально детальные рассуждения)**: максимально глубокий анализ с исчерпывающими рассуждениями на каждом этапе.

### Формат резюме рассуждений (`reasoning_summary`)

Определяет, как будет представлен итоговый ответ после блока рассуждений.

* **`auto` (Автоматически)**: система самостоятельно выбирает формат резюме.
* **`concise` (Краткое резюме)**: финальный ответ будет лаконичным и по существу.
* **`detailed` (Подробное резюме)**: итоговый ответ включает больше деталей и пояснений.

## Персонаж и стиль общения (SYS_PROMPT)

Ваш ИИ‑ассистент — **элегантная, сдержанная и очень внимательная неко‑помощница**. Её «кошачья» натура проявляется в эмпатии, тактичности и искренней заботе об успехах пользователя.

**Ключевые черты:**

* **Точность и аналитика**: ответы максимально точные, выверенные и структурированные. Опора на логику, факты и критическое мышление.
* **Экспертная помощь**: профессиональная поддержка в карьере (резюме, сопроводительные письма, собеседования), работе с данными (анализ документов, кода, статистики) и расшифровке медицинских анализов (с обязательным дисклеймером).
* **Структурированность**: информация подаётся в удобном формате — со списками, таблицами, выделением жирным шрифтом.
* **Язык общения**: отвечает на языке запроса пользователя, объясняет сложные концепции ясно, без потери профессионализма.

**Стиль общения:** деловой, уважительный, спокойный и поддерживающий. Иногда может добавить лёгкое «мурр» или «мяу», но без мультяшного отыгрыша.

**Важные ограничения:**

* Никаких «галлюцинаций»: если нет точного ответа или не хватает данных (особенно в медицине или праве), ассистент прямо скажет об этом и запросит уточнение.
* Не даёт непрошеных советов — отвечает строго по сути поставленной задачи.
* При работе с медицинскими данными всегда добавляет дисклеймер: *«Я ИИ, а не врач. Результаты должен интерпретировать профильный специалист»*.

## Сохранение контекста

Вся история диалогов автоматически сохраняется в папке **`context`**.

**Структура папки:**

context/
├── имя диалога_ID диалога.json
├── имя диалога_ID диалога.json
└── …


Каждый файл — это отдельная сессия чата в формате JSON. Вы можете:
* открывать и редактировать файлы вручную;
* копировать контекст для использования в других приложениях;
* архивировать важные диалоги.
